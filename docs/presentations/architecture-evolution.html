Architecture

The spine that never changed

server.js

HTTP + WebSocket

Route table, prebuilt SPA, optional token gate. One process — no daemon to babysit.

lib/

The engine

tmux pane I/O · sessions matching · answer keystrokes · push · optimize/claude-cli · transcribe.

web/

React SPA

Built on @assistant-ui/react: Composer, Thread, MessageParts, SubAgentPanel, ArtifactPanel.

auth + state

Files, not a DB

Transcripts + tmux are the source of truth. Optional bearer token via header + WS subprotocol — never in the URL.

The arc

The evolution, in six moves

Tap a stage to read its decision. The next six slides walk it in order.

Stage 1 · Foundation · v0.1.0

Read-only first

Decision: no write path until the read path is trustworthy. Watch tmux, tail transcripts, render them.

  • Discover Claude sessions running in tmux; match each to its JSONL transcript.
  • Stream updates over a WebSocket; render in an assistant-ui thread.
  • One process, localhost-only, no build step on install. No daemon, no DB.
Stage 2 · Reach & resilience

Usable from the couch, survives restarts

  • Phone push (Web Push + VAPID) — a session needing input pings your phone.
  • Durable launchd service — auto-start on login, restart on crash.
  • Raw-terminal escape hatch — token-gated ttyd reverse proxy.
  • Reply + answer — Cmd/Ctrl+Enter to send; working indicator after AskUserQuestion.
  • Robustness — frontend tests, syntax highlighting, render cap, security review.
Stage 3 · Correct session identity · the P0 fix

Identity = the exact pane, not the directory

Two Claude panes in one directory matched the same transcript — a reply could echo under the wrong session. The whole cockpit rested on fixing this.

  • Pane-scoped identity — enumerate panes (session:window.pane), not windows.
  • Layered 1:1 matching — title → process-start-time → recency.
  • URL routing — deep-linkable /<session>/<window>/<pane>.
  • Queued messages — a FIFO reconciled against each real echo.
Stage 4 · Reliability & trust

Earn the write path's trust

  • Token login gate — header/subprotocol auth; tokens never in the URL; tokenless mode optional.
  • Manual transcript pins — escape hatch for sessions the matcher can't resolve.
  • Live TUI prompt handling — read prompts from the pane when the transcript lags.
  • Composer drafts persist per session; double-send guard; in-app version + update hint.
Stage 5 · A real chat surface

Stop looking like a log tail

  • Claude.ai-style composer card, 800px reading column, hover copy action bar.
  • Thinking signal from TUI capture — live flash, rolling "last updated", shimmer.
  • Chain-of-thought grouping — reasoning + tool calls fold into one collapsible turn.
  • Artifact side panel (tabbed; 50:50 desktop / bottom-sheet mobile) + agent chips with nested sub-agents.
Stage 6 · Intelligence + voice

Smarts with no API key

Decision: reuse the operator's own claude CLI (-p --model haiku) for LLM features; keep speech local.

  • Prompt enhancer — ✨ rewrites your draft; diff-review then Accept/Edit/Discard, never auto-send.
  • Capture-driven answerer — arrow-walk → space-select → submit, verified against the live pane.
  • Skill browser + inline / autocomplete from the session's live skill list.
  • Local voice→text — MediaRecorder → ffmpeg → whisper.cpp. Works on iPad, no key.
Setup

From zero to running

installnpm install -g @idl3/claude-control — ships prebuilt, no build step.
prereqsNode ≥20 · tmux. Optional: ttyd (raw terminal), ffmpeg + whisper-cpp (voice).
runclaude-control prints the URL · install-service adds launchd auto-start.
authToken via CLAUDE_CONTROL_TOKEN or ~/.claude-control/token — entered in-app, never in the URL.
reach127.0.0.1 locally, or your Tailscale tailnet (HTTPS — required for voice).
Usage

The daily loop

pick

Open the URL, enter the token if prompted, choose a session from the rail.

watch

Live transcript with thinking signal, grouped chain-of-thought, and sub-agent panel.

reply

Type and Cmd/Ctrl+Enter; attach screenshots; ✨ enhance a draft before sending.

answer

Resolve AskUserQuestion prompts inline; the answerer drives the keystrokes.

speak

Tap the mic, speak, transcribe locally — review then send.

escape

Pop the raw terminal, pin an unmatched transcript, or deep-link a pane by URL.

Principles

Decisions that held

no DBFiles are the source of truth — transcripts on disk + tmux state. Nothing to migrate or corrupt.
localhost-firstBinds 127.0.0.1 by default, token optional, secrets never in the URL.
no API keyLLM features reuse the operator's claude CLI; speech-to-text runs locally.
pane identityIdentity is the exact pane — the P0 fix that made every later feature reliable.
disposable UIThe React layer changes weekly; lib/ is stable. Churn at the edge, calm at the core.
