The ask

“I want to speak my prompt
into the composer.”

A mic button and recording dialog already existed. But on the iPad it did nothing, and on desktop the transcription was unreliable.

Goal: tap mic → speak → text lands in the composer. Everywhere.

Diagnosis

Two bugs, not one

  • “Nothing happens” on tap — the live page was a stale cached SPA bundle. Deployed JS was newer than what the browser had loaded. (Exactly why pull-to-refresh exists.)
  • Silent transcription — the dialog used the Web Speech API, which is unsupported on iOS Safari and flaky on desktop Safari.

Lurking trap: getUserMedia needs a secure context (HTTPS/localhost). Over plain http:// Tailscale the mic is blocked entirely — we're on HTTPS, so fine.

Server · lib/transcribe.js

ffmpeg → whisper.cpp

export async function transcribe(inputPath, { lang = 'en' } = {}) {
  const ffmpeg  = resolveFfmpeg();
  const whisper = resolveWhisperBin();
  const model   = resolveWhisperModel();  // ~/.claude-control/models/ggml-base.en.bin
  // ...guard each (clear error if a binary/model is missing)...
  await run(ffmpeg,  ['-i', inputPath, '-ar','16000','-ac','1',
                     '-c:a','pcm_s16le','-f','wav', wav]);
  const { stdout } = await run(whisper, ['-m',model,'-f',wav,'-np','-nt','-l',lang]);
  return cleanTranscript(stdout);  // strip [BLANK_AUDIO], timestamps, blanks
}

The route mirrors the existing upload handler: byte-cap, temp file, token-gated. Binaries + model are env-overridable.

Client · VoiceDialog.tsx

Web Speech out, MediaRecorder in

rec.ondataavailable = (e) => { if (e.data.size) chunks.push(e.data); };
rec.start();                              // one blob, flushed on stop()

rec.onstop = async () => {
  const blob = new Blob(chunks, { type: mime });
  const text = await transcribeAudio(blob, ext);  // POST → server STT
  onCommit(text);                          // → composer (review-then-send)
};

Kept the waveform + Cancel/Pause/Stop. pickMime() selects webm/opus (Chrome) or mp4 (Safari) so iOS works too.

Why it works everywhere

One path, every device

  • Secure context: HTTPS over Tailscale → getUserMedia allowed.
  • Capture: MediaRecorder runs on Chrome, desktop Safari, and iOS Safari.
  • Transcription: on the Mac, not the browser — uniform across devices.
  • Privacy + cost: fully local whisper.cpp — no key, no audio leaves the host.
  • Always exitable: Cancel / Pause / Stop + full teardown — no stuck mic.

Takeaways

What this teaches

  • “Nothing happens” is often a caching problem, not a code problem.
  • Web Speech API ≠ cross-browser. For real coverage: record + server-STT.
  • The secure-context rule silently kills mic/camera over plain HTTP.
  • Keeping the waveform on Web Audio decoupled the UI from the STT engine.
